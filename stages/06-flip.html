Jargon on the front, plain terms on the back

Every Toolkit card serves two readers at once. The front lists the named tools for the person who evaluates stacks. The back translates the same work into outcomes for the person who signs off. One 0.72 second rotation moves between them. This is the Automation card, the first in the grid. Click it, or press Enter when it has focus.

perspective 1200px·flip 720ms quint-out· chips cascade 82ms·it is a real button
note / 01

Two faces, one grid cell

Both faces declare grid-area: 1/1, so they stack without absolute positioning and the card's height stays honest: it is always as tall as its tallest face. The back starts pre-rotated 180° and backface-visibility: hidden keeps each side invisible from behind.

note / 02

The turn feels physical

The button element carries perspective: 1200px, deep enough to read as an object turning rather than a texture warping. The 720ms quint-out curve spends its speed early and lands the back face gently, like a card set down rather than snapped.

note / 03

Chips arrive one at a time

On first reveal the seven tool chips cascade in with an 82ms stagger, each rising 11px out of a slight blur. The order matters: the reader scans Zapier before RAG, tools before concepts.

note / 04

It is a button, not a div

The card is a real <button> with aria-pressed tracking the flip. Enter and Space flip it, Escape restores the front, and flipping one card in the grid closes the others so only one back is ever open.
