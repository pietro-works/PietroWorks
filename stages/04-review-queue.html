stage_04 · highlight

Two lanes race the same task

The Architecture section backs its NDA banner with proof instead of a promise: the same review task run twice, side by side. On the left, a person opens a file, copies a value, mistypes it, and the row goes red. On the right, four pipeline stages ingest, map, verify and sync, and an approval stamp lands. Both lanes replay every 7.5 seconds, driven by one shared clock and a single synthetic cursor.

loop 7.5s·manual fails at 3.2s· automated approves at 6.75s·one shared cursor· typewriter 65ms/char ± jitter
note / 01

The failure is scripted to the keystroke

At 1.9s the manual lane starts typing 800 into font_weight, deletes it, and retypes 9OO, the letter O standing in for a zero. The typo is not incidental. It is why the row goes red at 3.2s: a human transcription error the automated lane structurally cannot make.

note / 02

One cursor plays both witnesses

The same #rqCursor element visits every row it needs to touch, in order, on a manually authored keyframe table. Between keyframes its position is interpolated with an ease-in-out curve, so it drifts like a hand rather than teleporting between clicks.

note / 03

Red and green share a grammar

Both lanes use the same row, badge and progress-bar components. Only the state classes differ: rq-lit while active, rq-done-green or rq-done-purple on success, rq-error with a glitch shake on failure. The comparison works because the visual language never changes, only the outcome.

note / 04

The AI step visibly thinks

Where the manual lane blocks on a red decision row, the automated lane's third stage shows three pulsing dots, a visibility: hidden element flipped visible only while the model drafts. It is the one moment on the site an AI step is shown working rather than instantly resolved.

note / 05

The stamp lands, then leaves cleanly

The approval stamp scales in at 6.75s, holds, then eases down to a smaller resting rotation before the loop resets it to zero for the next pass. A single <style> element also gets injected once at runtime to color every error badge, a small proof the component was built to be dropped in anywhere.
