A ghost operator builds the system

The Reliability section claims the tools are built from small modules, then proves it by performing the build. A synthetic cursor picks six files off the shelf, places each one as a glowing node, wires them in sequence, taps the finished cluster, and clears the canvas. Then it starts over, because a demo you can watch twice is a demo you trust.

note / 01

The cursor moves like a hand

Travel runs on a quint-out curve, the tap dips the pointer to 92% scale, and a ring pulses out from the hotspot. Between placements the cursor drifts 28% of the way toward its next target before committing. Nothing teleports, so the eye never loses the thread.

note / 02

Each module owns a hue

A per-file --module-rgb variable rides from shelf row to node tint to connector gradient. Four hues come from the brand spectrum; steel blue and amber extend it, because six distinguishable modules matter more here than palette purity.

note / 03

Placement depletes the shelf

The moment a file lands on the canvas, its shelf row fades to 40% and stops accepting the hover state. The inventory metaphor holds: a module is either on the shelf or in the machine, never both.

note / 04

The teardown radiates

Exit vectors point away from the canvas center, computed per node at placement time. Nodes blur as they scatter, connectors retract in reverse order, and the shelf relights in a 55ms cascade. The reset reads as breath out, not as a page refresh.

note / 05

The finished system gets acknowledged

After the last connector lands, the cursor travels to the cluster's measured centroid and taps once. Every node brightens with a green cast. It is a small beat, but it turns six placements into one system before the teardown begins.

note / 06

Motion respects the visitor

The whole performance sits behind an IntersectionObserver and a prefers-reduced-motion gate. Off screen it does not run. For visitors who opt out of motion, the board simply presents its shelf, which still makes the modularity argument on its own.
